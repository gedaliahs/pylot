Stop losing context when switching projects.

The problem

Every developer juggles multiple projects. Switching between them is a mess.

1

cd into the directory

Then remember which branch you were on. git checkout... which one was it?

2

Set up env vars

Re-source your .env file. Or was it .env.local? Did the API key change?

3

Start services

Spin up the dev server, database, worker. Oh wait, port 3000 is still used by the other project.

4

Remember what you were doing

What file were you editing? What was the bug? You've lost 10 minutes just getting back to where you were.

pylot does all of this in one command

How it works

Three concepts. That's it.

1

Save a context

When you're in a project directory, run pylot save <name>. pylot snapshots your current state: directory, git branch, env vars, and reads your .pylot.toml for services and ports.

~/api $ pylot save api
2

Switch between contexts

Run pylot switch <name> from anywhere. pylot will: cd to the project, checkout your branch, load env vars, start services, and kill port conflicts. All automatically.

~/mobile $ pylot switch api
3

Use the dashboard

Run pylot with no arguments to open an interactive TUI. Browse all your projects, see service health, port status, and switch with Enter.

~ $ pylot

What pylot tracks

Everything you need to resume working, nothing you don't.

📂

Project Directory

cd's you to the right place instantly

🌿

Git Branch

Checks out the branch you were on. Warns about uncommitted changes.

🔑

Env Variables

Loads your .env file and exports variables into the shell.

⚙️

Services

Starts dev servers, databases, workers. Stops them when you switch away.

🚢

Ports

Detects conflicts and offers to kill blocking processes before switching.

🕒

Last Accessed

Shows when you last worked on each project so you know where you left off.

Commands

Everything pylot can do.

Command
What it does
pylot save <name>
Save the current directory as a named context. Captures git branch, env vars, and reads .pylot.toml.
pylot switch <name>
Switch to a saved context. Restores directory, branch, env, starts services, resolves port conflicts.
pylot list
Show all saved contexts with their branch, service status, and last access time.
pylot status
Show the current directory's context: branch, env vars, dirty state, active ports.
pylot init
Generate a .pylot.toml config file in the current directory with smart defaults.
pylot stop [name]
Stop all running services for a context.
pylot remove <name>
Delete a saved context and stop its services.
pylot shell-init [shell]
Print the shell hook to add to your .zshrc / .bashrc / config.fish.
pylot
Open the interactive TUI dashboard. Navigate with arrow keys, switch with Enter.

Project config

Drop a .pylot.toml in your project root to define services and ports. Run pylot init to generate one automatically.

.pylot.toml 
# Which env file to load
env_file = ".env.development"

# Services to start when switching to this project
[services]
api    = "npm run dev"
db     = "docker compose up postgres -d"
worker = "npm run worker"

# Ports your project needs — pylot warns about conflicts
[ports]
required = [3000, 5432, 6379]

Setup

Two steps. Under a minute.

1. Install pylot

curl curl -fsSL https://raw.githubusercontent.com/gedaliahs/pylot/main/install.sh | sh
cargo cargo install --git https://github.com/gedaliahs/pylot
npm npm install -g pylot-cli

2. Add the shell hook

This lets pylot actually cd and export in your shell (without it, pylot can only print commands).

zsh eval "$(pylot shell-init zsh)" >> ~/.zshrc
bash eval "$(pylot shell-init bash)" >> ~/.bashrc
fish pylot shell-init fish >> ~/.config/fish/config.fish

3. Start using it

~/myproject $ pylot init         # creates .pylot.toml
~/myproject $ pylot save myproject # saves context
~/myproject $ pylot switch other   # switches away
~/other $ pylot                   # opens dashboard
